## How submission works

1. Each task has 3 radio buttons (OK / ERROR / Skipped) + a
   Comments textarea.
2. As you change anything, your progress auto-saves to your
   browser's local storage. Closing the tab is safe; reopening
   restores everything.
3. When you hit Submit at the bottom, the entire form posts as
   JSON to `https://yujin.app/api/v1/tester-reports/submit`.
4. The maintainer receives a summary email immediately.
5. If the endpoint is unreachable, the form falls back to opening
   your email client with the report pre-filled in the body.
6. You can also fill out the plain markdown / printable versions
   and email them manually -- the interactive form is just a
   convenience.

## Privacy

- We never see your tokens, your API keys, your real IP (only a
  one-way hash is stored for rate-limiting).
- Your email goes into the report ONLY IF you choose to type it
  in the sign-off block.
- All submissions are reviewed by the maintainer and used to
  iterate on the spec + the demos.

## Source of these forms

Same content as the markdown versions in this directory; the
HTML is generated by `tools/nac/render_tester_doc.py`. To
regenerate after editing the source `.md`:

```bash
python tools/nac/render_tester_doc.py path/to/checklist.md
```
