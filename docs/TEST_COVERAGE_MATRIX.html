NAC3 -- Test Coverage Matrix (automatic + manual)

Spec version: 2.2 + v2.3 preview. Generated: 2026-05-11. Authoritative for: the Yujin reference repository yujinapp/nac-spec on main.

This matrix lists EVERY artifact in the NAC3 ecosystem and reports its coverage by automated tests + the manual verification gate (the "human OK" checklist).

Adopters: copy this matrix structure for your own app. Replace columns with your artifacts; keep the same depth-by-row.

How to use this matrix

Per release

checklist](HUMAN_OK_CHECKLIST.md) is the executable form.

  1. Tag the spec version + reference suite version.
  2. Run bash tools/nac/test-launch.sh -- every AUTO row gates.
  3. Walk the MAN column -- the [Human OK
  4. Update the COVERAGE_REPORT_<date>.md with the run results.
  5. Adjust this matrix if the artifact landscape changed.

Per adopter

Copy this matrix structure for your own app. Replace artifact names; keep the same shape. The discipline is the same: every artifact gets an explicit auto + manual gate.

Anti-pattern

Do NOT mark an artifact "AUTO" if the test only checks file presence. AUTO means the test exercises behaviour. File-presence checks go under the harness (test-launch.sh), not the artifact matrix.
