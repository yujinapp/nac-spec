# NAC3 -- アクセシビリティへのコミットメント

**仕様バージョン:** 2.2 stable（+ v2.3 相互運用プレビュー）。
**最終レビュー日:** 2026-05-11。

NAC3 は、Web UI をマシンからアドレス指定可能にするために設計されました。AI エージェントが UI をナビゲートできるようにする特性は、スクリーンリーダー、スイッチデバイス、アイトラッカー、音声ユーザーにとっても同様に機能します。NAC3 は、設計上アクセシビリティのプリミティブであり、Yujin はそれを維持することをコミットします。

---

## コミットメントの内容

1. **WCAG 2.1 レベル AA** 準拠は、NAC3 を基盤とするすべての Yujin 製品（`yujin-pilot`、`yujin-forge`、yujin.app/nac-spec/ のリファレンスデモ、yujin.app/registry）における最低基準です。
2. **実現可能な範囲で AAA** を目指します。対象はアクセシビリティが特に重要な画面、すなわちチャットパネル、音声アクティベーション、初回オンボーディング、エラーメッセージです。
3. **「アクセシブル版」を別途提供しません**。アクセシビリティはメイン製品に同梱され、同一価格・同一リリースサイクルで提供されます。別エディションはユーザーを差別化し、品質の劣化を招きます。
4. **「後でアクセシブルにする」はしません**。各リリースは、[HUMAN_OK_CHECKLIST.md](HUMAN_OK_CHECKLIST.md) のセクション 8.6 および新設のスクリーンリーダースモークセクション（Track G7）に記載されたアクセシビリティチェックをゲート条件としています。

## サポートする支援技術

リファレンス実装は以下の環境でテストされています：

| AT カテゴリ | 検証済みツール |
|-------------|----------------|
| スクリーンリーダー | NVDA（Windows）、JAWS（Windows）、VoiceOver（macOS、iOS） |
| 音声コントロール | Yujin Pilot、Apple Voice Control、Windows Speech Recognition、Dragon NaturallySpeaking |
| スイッチアクセス | iOS Switch Control、Android Switch Access |
| アイトラッキング | Tobii Dynavox |
| 拡大表示 | ブラウザズーム最大 200%、ZoomText、macOS Zoom |
| キーボードのみ | 完全なキーボードナビゲーション、フォーカスの可視化、時間制限なし |

標準アクセシビリティツリー（ARIA、accessibilityRole、accessibilityLabel）を利用する AT はすべて NAC3 の恩恵を受けます。NAC3 要素は AT レイヤーが使用するのと同じセマンティック情報を持つためです。

---

## NAC3 がアクセシビリティに貢献する仕組み

- **安定した識別子（`data-nac-id`）**: スクリーンリーダーやスイッチアクセスは視覚的な位置に依存しません。識別子はデザイン変更後も維持されるため、AT ユーザーの操作の習熟も維持されます。
- **標準ロール（`data-nac-role`）**: ロールの列挙（action、field、tab など）は ARIA ロールと 1:1 で対応します。AT ユーザーはセマンティクス的に正確なアナウンスを受け取ります。
- **マニフェストの動詞（`label_i18n`）**: すべてのアクションに 10 言語のローカライズ済みラベルがあります。音声コントロールユーザーが動詞を発話すると、マニフェストがそれを解決します。
- **決定論的な確認イベント（`nac:action:succeeded`）**: AT ユーザーは UI アニメーションの推測ではなく、アクションが完了したという確認を受け取ります。
- **厳格なバリデーション（v2.2）**: マニフェストと DOM の乖離を AT ユーザーに届く前に検出します。

## NAC3 が解決しないこと

- **ネイティブ iOS/Android アプリケーション**: v2.2 仕様は Web + WebView のみを対象とします。ネイティブモバイルは v3.0 ロードマップに含まれています。
- **視覚的な表現**: NAC3 は構造的なものです。コントラスト、フォントサイズ、フォーカスインジケーターは実装側の責任です（Yujin のリファレンス実装では Yujin トークンがこれをカバーしています）。
- **複雑なフローの認知負荷**: NAC3 の ID は、設計の悪いワークフローをシンプルにするものではありません。それは優れた情報アーキテクチャと平易な言葉によるコピーが担います。
- **マルチメディアのキャプション**: 音声・動画アセットはパブリッシャーがキャプションを付ける必要があります。NAC3 はフックを提供しますが、コンテンツは提供しません。

## アクセシビリティの問題を報告する

`accessibility@yujin.app`（またはメンテナーに転送されるアドレス）にメールしてください。対応 SLA：トリアージまで 5 営業日。修正の SLA はケースごとに異なるため設定していません。問題は `nac-spec` リポジトリで `a11y` ラベルを付けて公開追跡されます。

セキュリティに関わる問題（例：確認ダイアログの AT バイパス）については、`SECURITY.md` に従ってください。

## なぜこれを公開するか

倫理的な理由に加え、実務上の理由が 2 つあります：

1. **EU Accessibility Act（EAA）** は 2025 年 6 月に B2C サービスへの適用が開始されました。Forge で構築されたアプリはデフォルトで NAC3 準拠となり、競合他社よりも EAA 準拠に近い状態で出荷されます。
2. **Web アプリに関する米国 ADA Title III 訴訟**は前年比 320% 増加しています。エンタープライズの購買担当者はこれを重視しており、NAC3 + Yujin のコンプライアンス体制は法的リスクを低減します。

NAC3 は「アクセシビリティをボーナスとして持つオープン標準」ではありません。NAC3 は「設計上アクセシビリティネイティブである唯一の汎用 Web 自動化コントラクト」です。私たちはそれを維持し続けます。
